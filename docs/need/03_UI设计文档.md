# AnimeProducer UI 设计文档

## 1. 设计目标

这套 UI 不应该再像通用后台系统，而应该像一个“动画制作工作台”：

1. 能表达阶段推进。
2. 能表达资产和产线状态。
3. 能同时容纳创意决策和批处理任务。

设计目标：

1. 用户一进入项目就知道当前阶段和下一步。
2. 复杂能力按阶段渐进展开，而不是一次性平铺。
3. 视觉风格体现“创作台 + 制作控制板”的气质。

## 2. 视觉方向

## 2.1 主题关键词

1. 分镜台
2. 制作板
3. 纸张感
4. 轨道感
5. 指挥台

## 2.2 视觉主题建议

建议采用 `Warm Studio Console` 方向，而不是继续走纯黑 SaaS：

1. 大面积使用暖白纸感底色，降低压迫感。
2. 用深石墨色承载导航和结构边界。
3. 用高识别度的橙红与蓝色区分“创作动作”和“AI/系统动作”。
4. 保留少量深色区域给时间线、预览器和媒体播放器。

这种方向更像“分镜台 + 制作指挥面板”，而不是普通数据后台。

## 2.3 色彩系统

建议色板：

1. `Canvas / 页面底色`: `#F6F1E8`
2. `Panel / 卡片底色`: `#FFFDF8`
3. `Panel Alt / 浅层分区`: `#EFE7D8`
4. `Ink / 主文字`: `#1F2430`
5. `Muted / 次级文字`: `#6A7280`
6. `Border / 分割边界`: `#D7CFBF`
7. `Primary / 主操作`: `#D95D39`
8. `Primary Hover`: `#BF4F2E`
9. `AI Accent / 智能动作`: `#2E6BFF`
10. `Success`: `#1D8F6A`
11. `Warning`: `#C68A12`
12. `Danger`: `#C4473D`

状态色建议：

1. `未开始`: 灰米色
2. `进行中`: 蓝色
3. `待审核`: 橙色
4. `已完成`: 绿色
5. `失败`: 红色
6. `阻塞`: 深橙红

## 2.4 字体系统

建议字体：

1. 标题：`Alibaba PuHuiTi 3.0 Heavy`
2. 正文：`HarmonyOS Sans SC`
3. 数据/编号：`JetBrains Mono`

字号层级：

1. Page Title: 32/40
2. Section Title: 22/30
3. Card Title: 16/24
4. Body: 14/22
5. Meta: 12/18

## 2.5 背景与质感

避免纯色平铺。

建议：

1. 页面主背景使用暖纸色渐变。
2. 大区块底层叠加极淡的网格纹理或 storyboard 标线。
3. 时间线与媒体区域使用深色底，形成明显工作区对比。

## 3. 信息架构

## 3.1 应用级结构

### 左侧主导航

1. 项目列表
2. 当前项目
3. 仪表盘
4. 书源与章节
5. 角色圣经
6. 分集规划
7. 分镜工作台
8. 渲染中心
9. 交付输出

### 顶部上下文栏

展示：

1. 当前项目名
2. 风格预设
3. 当前阶段
4. 全局搜索
5. 最近操作入口
6. 用户菜单

### 右侧通知/操作抽屉

统一承载：

1. 运行中任务
2. 最近完成任务
3. 失败提醒
4. 可点击跳转的恢复入口

## 3.2 页面关系

建议页面层级：

1. 项目列表页
2. 项目仪表盘
3. 书源与章节页
4. 角色圣经页
5. 分集规划页
6. 分镜工作台页
7. 渲染中心页
8. 交付输出页

不要再用一个页面容纳全部内容。

## 4. 页面详细设计

## 4.1 项目列表页

### 目标

让用户快速理解项目状态，并进入下一步最重要的项目。

### 布局

1. 顶部 Hero 区
2. 搜索与筛选区
3. 项目卡片瀑布或网格

### 卡片内容

每张项目卡必须显示：

1. 项目名
2. 风格标签
3. 当前阶段
4. 完成率
5. 最近更新时间
6. 下一步推荐动作

### 交互

1. 点击卡片进入仪表盘。
2. 卡片右上角菜单包含编辑、归档、删除。
3. 新建项目按钮放在页面右上角，视觉突出。

## 4.2 项目仪表盘

### 页面目标

回答三个问题：

1. 现在做到哪一步了？
2. 卡住没有？
3. 下一步该做什么？

### 页面结构

1. 项目头部区
2. 阶段总览区
3. 关键指标区
4. 最近操作区
5. 风险与阻塞区

### 项目头部区

内容：

1. 项目标题
2. 项目简介
3. 风格预设标签
4. 当前阶段徽章
5. 主按钮 `继续当前阶段`
6. 次按钮 `高级动作`

### 阶段总览区

使用 6 到 7 个横向阶段卡片：

1. 书源导入
2. 章节分析
3. 角色圣经
4. 分集规划
5. 分镜
6. 渲染
7. 交付

每张卡显示：

1. 状态图标
2. 进度条
3. 关键计数
4. 阻塞提示
5. 主入口按钮

### 关键指标区

建议四宫格：

1. 章节总数 / 已分析
2. 角色总数 / 已确认
3. 分集总数 / 已出分镜
4. 渲染批次 / 已完成

### 最近操作区

使用纵向时间线卡片：

1. 操作类型
2. 当前状态
3. 进度
4. 发生时间
5. 跳转按钮

### 阻塞区

单独用高对比色卡片展示：

1. 待确认角色锚点图
2. 某集尚未审核 storyboard
3. 渲染失败批次

## 4.3 书源与章节页

### 页面目标

让用户完成“上传原著 -> 看懂解析结果 -> 修正文档结构”。

### 页面布局

分成左右两栏：

1. 左侧：书源信息与上传区
2. 右侧：章节目录与详情

### 左侧区域

模块：

1. 书籍元数据卡
2. 上传拖拽区
3. 解析统计
4. 全书分析摘要
5. 批量章节分析按钮

### 右侧区域

模块：

1. 章节筛选工具条
2. 章节列表
3. 章节详情抽屉

### 章节列表项

每一项显示：

1. 章节号
2. 标题
3. 字数
4. 分析状态
5. 情绪标签
6. 关键事件摘要
7. 建议归属分集

### 交互细节

1. 点击章节打开右侧详情抽屉，不建议再用居中弹窗。
2. 详情抽屉支持单章分析、编辑、删除。
3. 支持多选章节后执行 `批量分析`、`扫描角色`。

## 4.4 角色圣经页

### 页面目标

这是产品的核心差异页。  
它必须让用户感到这不是“角色表”，而是真正的角色资产台。

### 页面布局

建议三栏布局：

1. 左栏：角色列表与筛选
2. 中栏：角色信息与图像墙
3. 右栏：待办与审核面板

### 左栏

内容：

1. 搜索框
2. 状态筛选
3. 角色列表
4. 角色来源章节标签

角色列表项显示：

1. 头像缩略图
2. 名称
3. 锚点状态
4. 语音状态
5. 风险提示

### 中栏

显示当前角色详情：

1. 大图预览
2. 角色描述
3. 别名
4. 出场章节
5. 语音配置
6. 候选图图墙
7. 变体图图墙

### 右栏

显示：

1. 当前角色待处理事项
2. AI 识别出的别名冲突
3. 疑似身份重复提醒
4. 推荐动作

### 关键交互

1. 候选图 hover 显示 `设为锚点图`。
2. 锚点图固定置顶显示。
3. 合并角色、添加别名等风险操作走侧边确认面板，不用轻量 toast。
4. 语音试听区域与角色图像管理放在同一上下文内。

## 4.5 分集规划页

### 页面目标

帮助用户从章节过渡到正式生产单元。

### 页面布局

建议上下结构：

1. 上部：规划参数和 AI 规划入口
2. 中部：规划草案时间轴
3. 下部：章节覆盖关系图

### 规划草案卡片

每张卡片显示：

1. 集数
2. 标题
3. 章节范围
4. 预计时长
5. 核心剧情摘要
6. 关键事件
7. 操作按钮 `确认本集` 或 `编辑`

### 交互

1. 生成草案后默认不直接落库。
2. 用户可以整版确认，也可逐集微调后确认。
3. 删除旧分集时给出影响提示。

### 视觉表现

建议加入“章节条带图”：

1. 横轴按章节编号排列。
2. 每集用不同色块覆盖区间。
3. 这样用户能直观看到分集切分是否均衡。

## 4.6 分镜工作台

### 页面目标

让用户从“列表看镜头”升级为“在时序和结构上看镜头”。

### 页面布局

建议三栏：

1. 左栏：分集列表
2. 中栏：镜头轨道/镜头序列
3. 右栏：镜头详情与审核

### 左栏

每个分集项显示：

1. 集数和标题
2. storyboard 状态
3. shot 数量
4. 是否待审核

### 中栏

用卡片式时间线展示镜头：

1. 镜头编号
2. 时长
3. 场景说明
4. 出场角色标签
5. 台词标记
6. 渲染状态占位

不要只做普通列表，应该做成“轨道式阅读体验”。

### 右栏

展示当前镜头详情：

1. scene description
2. visual prompt
3. camera movement
4. 角色
5. 台词
6. 备注
7. 审核动作

### 交互

1. 生成 storyboard 按钮只出现在未生成分镜的分集上下文里。
2. 审核通过后，页面显式显示“可以启动渲染”。
3. 镜头之间支持键盘上下切换，提高审核效率。

## 4.7 渲染中心

### 页面目标

让渲染过程从“后台黑盒”变成“可追踪生产批次”。

### 页面布局

分成三块：

1. 运行中批次
2. 失败批次
3. 已完成批次

### 批次卡片

显示：

1. 对应分集
2. 当前状态
3. 进度条
4. 已完成镜头数 / 总镜头数
5. 最近错误
6. 操作按钮

### 批次详情抽屉

打开后展示：

1. 镜头级渲染列表
2. 每个镜头的状态
3. 失败原因
4. 重新渲染入口

### 交互

1. 运行中批次固定置顶。
2. 失败批次高亮。
3. 批次完成后自动在交付页生成一条新记录。

## 4.8 交付输出页

### 页面目标

把“输出视频路径”变成真正可管理的交付区。

### 页面布局

1. 顶部最新版本大播放器
2. 下方版本列表
3. 右侧版本元信息

### 版本卡片显示

1. 分集名
2. 生成时间
3. 时长
4. 来源渲染批次
5. 下载/复制链接按钮

## 5. 通用组件规范

## 5.1 阶段卡

用于仪表盘和各页顶部。

结构：

1. 标题
2. 状态图标
3. 进度
4. 指标
5. 阻塞提示
6. 主按钮

## 5.2 操作卡

用于统一展示 operation/job。

结构：

1. 操作名
2. 状态
3. 进度
4. 目标对象
5. 时间
6. 取消/重试/查看详情

## 5.3 状态徽章

分两类：

1. 生产状态徽章
2. 审核状态徽章

不要混用一个颜色系统表示所有状态。

## 5.4 详情抽屉优先于弹窗

建议规则：

1. 查看详情用抽屉。
2. 轻量确认用弹窗。
3. 复杂编辑用独立页或抽屉。

## 6. 交互原则

## 6.1 页面必须有唯一主动作

每个页面顶部只能有一个最强主按钮。

例如：

1. 书源页：`上传并解析书籍`
2. 角色页：`扫描角色` 或 `继续确认角色`
3. 分集页：`生成分集草案`
4. 分镜页：`生成本集分镜`
5. 渲染页：`启动渲染`

## 6.2 按阶段显示动作

不应该让用户同时看到 5 个同权重按钮。  
动作必须根据上下文显隐。

## 6.3 长任务必须有四层反馈

1. 提交成功反馈
2. 持续进度反馈
3. 完成结果反馈
4. 失败恢复反馈

## 6.4 失败不是 toast 结束

失败必须有可恢复入口：

1. 查看原因
2. 重试
3. 跳转到相关对象

## 6.5 用语言替代技术词

建议前端文案转换：

1. `build assets` -> `生成角色资产`
2. `generate storyboard` -> `生成分镜`
3. `start pipeline` -> `启动渲染`
4. `shot render` -> `镜头渲染`

## 7. 响应式策略

## 桌面端

优先展示三栏与工作台布局。

## 平板端

左栏可折叠成抽屉。

## 手机端

以摘要页和列表页为主，不建议在移动端承担完整分镜审核工作。

移动端保留：

1. 仪表盘
2. 章节查看
3. 角色查看
4. 渲染状态跟踪
5. 输出预览

## 8. 设计交付建议

如果你后续交给 UI/UX 设计师，可以按以下顺序出稿：

1. 先出应用壳层和仪表盘。
2. 再出书源与章节页、角色圣经页。
3. 再出分集规划页、分镜工作台页。
4. 最后补渲染中心和交付页。

建议设计稿至少包含：

1. 桌面端高保真
2. 平板端适配
3. 关键组件库
4. 状态体系说明
5. 页面间跳转关系

## 9. 一句话设计基调

AnimeProducer 的 UI 应该像“动画导演的生产台”，不是“通用后台管理系统”。
